# Changelog

## 2026-03-04 — Maintenance & fixes

- Fan / PWM (RELAY_FAN) conflict — applied FIX Option A
  - Problem: `RELAY_FAN` (GPIO 13) was used both for LEDC PWM and digital
    writes; `digitalWrite()` detached LEDC and broke PWM. `gRelayState.fanOn`
    was never updated by PWM path so telemetry reported `fan=0`.
  - Change: removed `RELAY_FAN` handling from `setRelay()` and replaced
    direct digital write in `allRelaysOff()` with a call to `setFanSpeed(0)`.
    `setFanSpeed()` now updates `gRelayState.fanOn` under `controlMutex`.
  - Files changed: [globals.h](globals.h), [globals.cpp](globals.cpp),
    [task_incubator.cpp](task_incubator.cpp)
  - Test: project compiled successfully; run on device and telemetry now
    reports fan state according to PWM value.

- Turner last-turn timestamp correction
  - Problem: `lastTurnEpoch` was saved using the epoch read before the turn
    action, causing the reported idle interval to be (interval - duration).
  - Change: after the turn completes, firmware now re-reads the RTC and
    persists the post-turn epoch as `lastTurn` in NVS.
  - Files changed: [task_incubator.cpp](task_incubator.cpp)
  - Test: code compiled; recommend running device and verifying turn timestamp
    reported on next telemetry/after reboot.

- Cloud secrets and deployment handling
  - Change: moved cloud endpoint and token to a local gitignored `secrets.h`;
    added `secrets.h.example` with placeholders and added `.gitignore` entry.
  - Also updated `config.h` to optionally `#include "secrets.h"` when present
    (safe empty defaults otherwise).
  - Files changed: [config.h](config.h), [secrets.h.example](secrets.h.example),
    [secrets.h](secrets.h), [.gitignore](.gitignore)
  - Test: compiled firmware; cloud telemetry continues to upload (HTTP 200)
    when `secrets.h` contains a valid URL + token.
